Generated Binary Search Tree (BST) 🌴

To explain how this works, each node is an object holding a value, and two branches: left & right.
If a node has no child on a particular side, it'll return as :null.

If a new number is smaller than its current node's value, it'll branch off to the left. However, if it's
larger, it veers to the right instead.

So in saying that, by inserting 37, 17, and 209 in that specific order you'll see that:

  • 37 becomes the root.
  • 17 is less than 37 in this case, so it becomes the left child.
  • 209 is greater than 37, becoming the right child!

An easy way to put it is that this structure makes searching for values easy, due to how every step the tree
makes, it decides the direction to go (left for smaller, right for bigger!)
